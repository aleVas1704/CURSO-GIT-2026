## apuntes clase 8

### branch naming pattern

es como una regla para poner nombres a las ramas. 
no podemos poner cualquier cosa, tiene que decir qué estamos haciendo (ej: update-readme, fix-login).

sirve para que todo esté ordenado y los demás entiendan rápido qué hace cada rama.


### pull request (PR)

es básicamente pedir permiso para unir tu rama con otra 
no se mergea directo, primero alguien revisa tus cambios.

sirve para evitar errores y tener control de lo que se sube.

### revision de PRs

no solo hacer nuestra parte, también debemos revisar la de otros. 
ver cambios y opinar si es god o zzz

### reglas y permisos en github

se pueden poner reglas para que nadie haga cambios directos en ramas importantes.  
todo tiene que pasar por PR sí o sí.

esto hace el repo más seguro y ordenado.

### git stash

sirve para guardar cambios temporalmente sin hacer commit. 

### git stash pop

recupera lo que se guardo con stash. 

### git diff

muestra qué cambiaste antes de hacer commit. 
Ahi podemos ver  líneas agregadas, borradas o modificadas.

sirve para no subir cosas que no queriamos

### git commit con editor
esto mas que todo para hacernos recordar de la 3ra clase lo del commit directamente
y hacer una descripcion mas larga

### visual studio code y git

vs code nos permite ver cambios y resolver conflictos fácil.
